# 🍽️ Food Ordering App (iOS, SwiftUI)

Мобильное приложение для заказа еды с поддержкой карты, регистрации, корзины, локализации и автоматической сборки через GitHub Actions.

---

## Description

Food Ordering App — это мобильное приложение, разработанное в рамках лабораторной работы №9 по дисциплине «Технологии программирования для мобильных приложений».

Приложение предоставляет пользователю следующие возможности:

- регистрация и вход в аккаунт;
- выбор ближайшего ресторана автоматически или на карте;
- просмотр категорий блюд и добавление их в корзину;
- изменение количества блюд и добавление комментария к заказу;
- выбор способа оплаты (онлайн, ЕРИП, терминал, наличные);
- указание адреса доставки;
- оформление заказа;
- локальное хранение данных (SQLite + UserDefaults).

Проект реализован на **SwiftUI**, с архитектурой **MVVM**, поддержкой **CoreLocation**, **MapKit**, **SQLite/CoreData**, локализацией на 3 языка, Unit/UI тестами и CI/CD через GitHub Actions.

## Installation

Чтобы запустить проект локально:

```bash
git clone https://github.com/tatttti/food-ordering-app-ios.git
cd food-ordering-app-ios
git checkout development
open FoodOrderingApp.xcodeproj
```
Требования:

- macOS 13+

- Xcode 15+

- iOS 16+

## Usage


После запуска приложения пользователь может:

1. Зарегистрироваться или войти (данные сохраняются через UserDefaults).
2. Выбрать ресторан:
   - автоматически ближайший (CoreLocation),
   - или вручную на карте (MapKit).
3. Просматривать блюда и добавлять их в корзину.
4. Изменять количество блюд, добавлять комментарий.
5. Указывать адрес доставки и выбирать способ оплаты.
6. Подтверждать заказ (данные сохраняются в SQLite/CoreData).

## Branching Strategy

### Основные ветки

- main — стабильная версия  
- production — релизная версия  
- development — активная разработка  

### Правила работы

- Все изменения выполняются только в `development`.
- Каждая задача — отдельная feature‑ветка:
  - feature/ui
  - feature/coredata
  - feature/tests
  - feature/map
  - feature/<task-name>
- После завершения работы создаётся Pull Request → Review → Merge.

### Защита веток

- Прямые пуши в main и production запрещены.
- Force push запрещён.
- Удаление веток запрещено.
- Linear history включён.
- Изменения только через Pull Request.

## CI/CD (GitHub Actions)

GitHub Actions используется для:

- автоматической сборки iOS‑проекта;
- запуска Unit/UI тестов;
- проверки Pull Request;
- запрета merge при ошибках тестов.

Workflow расположен в:
```bash
.github/workflows/ios-ci.yml
```
